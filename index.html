GUIA DE ACESSIBILIDADE HTML PARA DEFICIÊNTES VISUAIS

Um ponto pouco questionado na internet é a acessibilidade, que acaba sendo deixada de lado por diversos motivos na hora de desenvolver um site em HTML.

De acordo com um levantamento feito pelo Movimento Web Para Todos e a plataforma de dados BigDataCorp, divulgado no dia 20 de maio do ano de 2020, a maior parte da web brasileira é inacessível aos 45 milhões de cidadãos que possuem alguma deficiência. Os dados são assustadores: menos de 1% dos sites tiveram sucesso em todos os testes de acessibilidade aplicados.

Sendo assim trago algumas soluções para mudar esse cenário, com exemplos de código, principalmente para desenvolvedores front-end implentarem nos seus códigos e mudarem esse cenário web.

A linguagem do seu documento

Logo no início do HTML, defina a linguagem que você está utilizando. A definição do idioma no documento ajuda tecnologias assistivas a definirem o perfil de voz e conjunto de caracteres.

Usando o atributo lang da própria tag ‹html›, como mostra o exemplo:

          
‹!DOCTYPE html›
‹html lang="pt-br"›

  ‹head›
    ‹!-- Cabeçalho da página --›
  ‹/head›

  ‹body›
    ‹!-- Corpo da página --›
  ‹/body›
‹/html›

Estrutura textual semântica

Utilize tags adequadas para títulos(‹h1›), subtítulos(‹h2› ao ‹h6›), parágrafos (‹p›) e listas (‹ul› ou ‹ol›) no seu conteúdo textual. O leitor de telas vai identificar cada elemento para o usuário, permitindo ainda navegar de título em título.

Exemplo de uma estrutura acessível de um conteúdo textual é:

          
‹div id="titulo"›Titulo‹/div›
‹br›‹/br›
Texto sem marcação e nenhuma acessibilidade.
‹br›‹/br›
1.item da lista
‹br›‹/br›
2.item da lista
‹br›‹/br›
3.item da lista

Esse exemplo terá a mesma aparência visual, porém não terá a mesma semântica:

          
‹h1›Titulo‹/h1›

‹p›Texto com marcações e com acessibilidade‹/p›

‹ol›
  ‹li›item da lista‹/li›

  ‹li›item da lista‹/li›

  ‹li›item da lista‹/li›
‹/ol›

Elementos de interação

Controles de User Interface (UI) são os elementos com os quais o usuário interage, como botões, links e formulários. Navegadores permitem que esses controles sejam acessados pelo teclado. Eles devem ser usados adequadamente, pois são elementos com padrões de acessibilidade nativos que facilitam a navegação.

Landmarks

Landmarks são pontos de referência utilizados por leitores de tela para saltos no conteúdo. Devem ser usados para marcar blocos importantes, como menu de navegação, conteúdo principal, campos de busca, entre outros, permitindo que o usuário chegue diretamente a esses elementos.

Para criar landmarks, devemos incluir o atributo ‹role› no elemento, identificando seu valor semântico (como search, form, navigation, article, entre outros).

          
‹div role="navigation"›
  ‹h2›Papel Menu‹/h2›
  ‹ul role="menu" id="menu"›
    ‹li role="menuitem"›‹a href="#">MenuItem 1‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 2‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 3‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 4‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 5‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 6‹/a›‹/li›
    ‹li role="menuitem"›‹a href="#">MenuItem 7‹/a›‹/li›
  ‹/ul›
‹/div›

Imagens

No HTML, as imagens devem possuir o atributo alt com um texto alternativo que descreva seu conteúdo. Para assim o leitor de tela poder fazer seu trabalho.

            
‹img src="imagem/foto_de_cachorro" alt="foto ilustrando um pug tomando água"‹/img›

Tabelas

Tabelas devem ser escritas em HTML com células de cabeçalho associadas corretamente às suas colunas ou linhas, para uma leitura mais clara do leitor de telas. Ou seja, é importante separar corretamente o conteúdo em tags ‹th› e ‹td›. É possível, ainda, especificar se é cabeçalho de linhas ou colunas, utilizando o atributo scope. 

             
‹table›
  ‹tr›
    ‹th›‹/th›
    ‹th scope="col"›Month‹/th›
    ‹th scope="col"›Savings‹/th›
  ‹/tr›
  ‹tr›
    ‹td›1‹/td›
    ‹td›January‹/td›
    ‹td›$100‹/td›
  ‹/tr›
  ‹tr›
    ‹td›2‹/td›
    ‹td›February‹/td›
    ‹td›$80‹/td›
  ‹/tr›
‹/table›
